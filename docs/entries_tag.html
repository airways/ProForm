Entries Tag (BETA)

The Entries Tag provides a way to query and display any entry created through ProForm.

Beta This module is currently in a public beta phase, but has been used on multiple sites in an internal beta for several months. Any issues should be filed with the developer at the Devot:ee site for ProForm and will be fixed promptly.

Here's a simple example of using the Entries Tag to display all entries created in a particular form:

{exp:proform:entries form_name="{segment_2}"}
    <tr>
        <th>Row Number</th>
        <th>Form Entry ID</th>
        <th>User Agent</th>
        <th>IP Address</th>
        {fields}
        <th>{field_label}</th>
        {/fields}
    </tr>

    {if entries:no_results}
    <tr>
       <td colspan="100" class="error">No entries!</td>
    </tr>
    {if:else}
    <tr>
        <td>{row:number}</td>
        <td>{value:form_entry_id}</td>
        <td>{value:user_agent}</td>
        <td>{value:ip_address}</td>
        {fields}
            {if field_type == 'file'}
                {if field_value}
                    <td><img src="{field_value}" /></td>
                {if:else}
                    <td></td>
                {/if}
            {if:else}
                <td>{field_value}</td>
            {/if}
        {/fields}
    </tr>
    {/if}
{/exp:proform:entries}

Parameters

The following parameters are available:

  • entry_id="1"

Single Variables

The following variables are available in a notification template:

  • {form_id}
  • {form_label}
  • {form_name}
  • {value:form_entry_id}
  • {value:user_agent}
  • {value:ip_address}

{form_id}

Provides the unique numeric ID of the form.

{form_label}

Provides the firendly form label for the form. This can be any human readable string.

{form_name}

Provides the internal form name for the form. This should be all lowercase and only contained letters, numbers, and underscores.

Variable Pairs

The following variables are available in a notification template:

  • {fields}
  • {fieldrows}
